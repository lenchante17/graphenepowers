# Small Task Lightweight Example

This example shows how the canonical workflow looks for a `Small Task` that does not need windowing.

## Files

- `execution-plan.md` — lightweight plan with explicit task graph
- `plan-progress.md` — machine record plus human views for a non-windowed task

## What This Example Demonstrates

- `planning_mode: lightweight`
- no current-window section
- blockers still remain visible
- review state still remains explicit
